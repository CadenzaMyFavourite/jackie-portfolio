Front-end & UI Frameworks

This page describes the UI frameworks I rely on and how they shape the structure and appearance of my applications.

TailwindCSS

Tailwind’s utility classes let me build responsive layouts quickly without leaving HTML. Instead of writing custom CSS, I compose classes for spacing, typography, colors, and responsive behavior.

**Example:** the entire portfolio site uses Tailwind for layout and theming; dark mode toggling is managed with a single `dark` class on the `` element.

Workflow: configure `tailwind.config.js` with custom colors or fonts, use `@apply` for shared styles, and purge unused classes during build to keep CSS small.

PySide6

For Python desktop tools, PySide6 provides access to Qt’s widgets and graphics. It’s great for internal tooling where a local GUI is more convenient than a web app.

Typical project flow: design UI with Qt Designer, generate Python bindings, and implement business logic in separate modules. Packaging with PyInstaller creates stand‑alone executables.

React (future)

React is on my roadmap for more dynamic frontends. Its component-based architecture and hooks simplify state management compared to vanilla JavaScript.

I plan to use Create React App or Vite to bootstrap new projects and integrate TypeScript for type safety. This will enable richer interactivity in dashboards and admin panels.
